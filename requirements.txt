我要做一个Project, 关于Bahasa Isyarat Malaysia BIM 的教学产品. 这个产品是独立的, 他无需网络便可运行, 能带去任何地点. 
它使用的材料有, respberrypi4 model B, raspberry PI 7inch touchscreen, HD webcam, speacker 和power supply. 
我的产品需要能顺畅地在respberry pi 4 运行, 没有任何卡顿. 

首先, 我需要有一个学习页面展示在touchscreen里, 页面中能够实时展示内容不卡顿. 
一开始开机后页面会是在选项页面, 他是一个目录来的, 使用者能够选择学习的单词, 很多单词时会有下滑功能. 
当使用者选定特定的单词后,就会跳转到该词汇的学习内容. 这个页面右上角显示现在学习的手语的名称不用太大(比如Hello).
然后中间荧幕的左半显示手势的照片/动作的教学视频, 而右半实时显示使用者现在的动作.接着下方有一些按键.
第一个按键是退出按钮, 按了后会回到主页面, 使用者能够重新选择他要学习的单词, 按了后就会跳转到对应的学习内容. 
第二个按键是左半的教学视频重播按钮(如果是视频的话, 图片就不用了)(一开始教学视频只会播放一遍). 
这些就是第一开始的学习页面要求.

接着是学习内容的设计. 
首先学习内容时逐步教学的, 一步一步学习, 比如学习Hello手语时, 第一步会先展示手语的手部握法的图片在左侧, 而右侧会实时显示和检查使用者.
使用者需要做出对应正确的手势(手部动作), 并且维持住这个手势2秒以上(会显示青色的进度条), 才能算正确. 
首部动作错误时, 显示"动作错误", 正确后累积进度条, 满后显示恭喜正确!!.
接着如果教学没有下文了(简单的动作比如A, B, C,...),便显示要再次学习/主页面, 给使用者选择. 
如果学习还有下文(有特定手部放置位子的需求),便自动到下一步, 下一步可能学习手部放置位子, 比如放在头上, 腰上或者胸上. 一样的流程, 验证2秒等等. 
如果还有第3步骤, 有移动的需求就到第3部, 因为是连贯动作需求, 就不会有检测进度条了, 而是动作检测正确性, 需要做正确动作3次, 才算正确.
完成后一样, 会问是否回到主页/再来一次.

def keyPressEvent(self, event):
        """
        🚀 【新增核心监听】：当在当前学习/测试页面时，按键触发数据采集
        """
        # 如果当前进来的不是 "Test" 测试模式，则不触发任何逻辑，交还给系统默认处理
        if self.current_lesson != "Test":
            super().keyPressEvent(event)
            return

        # 🎯 设定触发按键：这里以键盘上的 [空格键] 为例（Qt.Key_Space）
        # 你也可以根据喜好改成 [Enter键](Qt.Key_Return) 或 字母键 [D键](Qt.Key_D)
        if event.key() == Qt.Key_Space:
            # 安全检查：确保后台视觉引擎不仅初始化了，而且里面的手势实例确实存在
            if hasattr(self, 'vision_engine') and self.vision_engine.gesture_instance:
                # 检查当前的实例是不是我们写的那个 TEST 实例
                if self.vision_engine.gesture_instance.name == "TEST":
                    print("\n[UI提示] 📥 成功捕获按键！正在导出当前帧手部几何分析数据...")
                    # 👈 啪！远程开锁！让下一帧在终端打印出来
                    self.vision_engine.gesture_instance.trigger_capture() 
        else:
            # 如果按的是别的键，原封不动抛回给 PyQt 默认机制
            super().keyPressEvent(event)

# 🌟 将 run() 方法中的摄像头启动修改为：
def run(self):
    self._is_running = True
    # 树莓派不需要 CAP_DSHOW
    self.cap = cv2.VideoCapture(0) 
    
    # 🌟 强制将硬件输入分辨率压缩至 320x240 或 640x480
    # 这能帮树莓派节省 70% 像素点的密集矩阵计算，FPS 直接飙升
    self.cap.set(cv2.CAP_PROP_FRAME_WIDTH, 640)
    self.cap.set(cv2.CAP_PROP_FRAME_HEIGHT, 480)
